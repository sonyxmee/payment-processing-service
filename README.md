# Async Payment Processing Service

Микросервис для асинхронной обработки платежей, построенный на принципах отказоустойчивости и гарантированной доставки сообщений.

## Стек технологий
* **Python 3.14+** (менеджер пакетов `uv`)
* **FastAPI** — веб-фреймворк
* **SQLAlchemy 2.0** — асинхронная работа с БД
* **PostgreSQL** — основное хранилище
* **RabbitMQ** — брокер сообщений
* **Docker & Docker Compose** — контейнеризация

## Ключевые возможности
* **Transactional Outbox Pattern** для обеспечения целостности данных между БД и брокером.
* **Идемпотентность** запросов через `Idempotency-Key`.
* **Retry Logic** с экспоненциальной задержкой.
* **Dead Letter Queue (DLQ)** для обработки критических ошибок.

## Быстрый старт

### 1. Клонирование репозитория
```bash
git clone <ссылка_на_репозиторий>
cd payment
```

### 2. Подготовка окружения
Создайте файл `.env` из примера и заполните его необходимыми учетными данными:
```bash
cp .env.example .env
```
*Убедитесь, что все переменные в `.env` заполнены корректно (настройки подключения к БД, RabbitMQ и API ключ).*

### 3. Запуск инфраструктуры
Для развертывания всех сервисов (API, Consumer, PostgreSQL, RabbitMQ) используйте Docker Compose:
```bash
docker-compose up --build -d
```

### 4. Применение миграций
Примените миграции базы данных, чтобы создать структуру таблиц:
```bash
docker-compose exec api alembic upgrade head
```
*Сервис будет доступен по адресу `http://localhost:8000`.*

## Локальная разработка (без Docker)

Для локальной работы используйте команды `make`. Они автоматически используют `uv` для управления окружением.

### Список доступных команд:
- `make dev` — запуск API в режиме разработки с hot-reload (`uvicorn`).
- `make lint` — запуск линтера `ruff` для проверки стиля и ошибок.
- `make format` — автоматическое приведение кода к формату `ruff`.
- `make sync` — синхронизация виртуального окружения и установка зависимостей.

1. **Инициализация проекта** (если еще не сделано):
   ```bash
   uv init payment
   uv venv --python /usr/bin/python3.14
   uv python pin /usr/bin/python3.14
   ```

2. **Установка зависимостей**:
   ```bash
   uv sync
   ```

3. **Запуск приложения**:
   ```bash
   uv run python -m application.main
   ```
